## Infrastructure Fixes

### Bug 1: `performance_metrics` always null
**Root cause:** `analyze_market_data()` in `src/data/indicators.py` did not expose a `'returns'` key. `daily_run.py` checked `if 'returns' in hist_data` but the key was missing, so `position_returns` was always empty.

**Fix:** Added `"returns": df_with_indicators['Daily_Return'].dropna().tolist()` to the asset dict in `analyze_market_data()`.

### Bug 2: CVaR tail_risk_analysis ValueError
**Root cause:** Different assets from yfinance can have slightly different history lengths (29 vs 28 days). `daily_run.py` passed raw arrays of mismatched lengths to `tail_risk_analysis()`, causing `ValueError: operands could not be broadcast together with shapes (29,) (28,)`.

**Fix:** Aligned returns to `min_len` before computing the weighted portfolio return series for `tail_risk_analysis()`, consistent with how `calculate_portfolio_cvar()` already handled it internally.

**Tests:** 211 passed, 3 warnings (pre-existing regime detector edge cases).

**Commit:** `7d1360f` on `dev`, pushed to `origin/dev`.

### Remaining Issue: Beta/Alpha still null
Even after the fix, `beta` and `alpha` remain `null` in results because the aligned portfolio return series is often < 30 days (yfinance quirks shorten the usable history). `calculate_all_metrics()` requires `len(returns) >= 30` for benchmark-relative metrics. This is statistically sound but may need a lower threshold or a longer fetch period for small portfolios.
