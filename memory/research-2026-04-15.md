## Development: Partial Sell Orders

### Problem
The LLM trading agent occasionally recommends **partial sells** (e.g., "sell 30% of DSY.PA"), but `Portfolio.sell()` only supported all-or-nothing liquidation. This created a divergence between LLM intent and system execution — noted explicitly in yesterday's memory when DSY.PA was sold entirely despite a 30% partial recommendation.

### Solution
Implemented proportional position reduction in `src/portfolio/portfolio.py`:

1. **`sell()` now accepts `pct: Optional[float]`**
   - `None` or `100.0` → full liquidation (backward compatible)
   - `0 < pct < 100` → sell exactly that percentage of shares

2. **P&L accounting**
   - Realized PnL computed proportionally: `quantity_sold * (price - avg_price)`
   - `avg_price` is preserved for the remaining position (cost basis unchanged)
   - `total_realized_pnl` incremented correctly

3. **Caller updates**
   - `src/daily_run.py` now passes `pct` through to `sell()`
   - `src/backtest/backtest.py` updated similarly for consistency
