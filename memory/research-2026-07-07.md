## Codebase Work

### Fix : `calculate_beta_alpha` NaN guard

**Fichier :** `src/risk/performance_metrics.py`  
**Problème :** Le dénominateur de la beta utilisait `benchmark_variance == 0 or benchmark_variance < 1e-15`, sans garde `np.isnan`. Si la variance du benchmark est `NaN` (série constante avec des NaN, par exemple), le code diviserait par `NaN` et propagerait des valeurs invalides.

**Fix :**
```python
if abs(benchmark_variance) < 1e-15 or np.isnan(benchmark_variance):
    return None, None
```

Cela aligne `calculate_beta_alpha` sur le pattern défensif déjà utilisé dans `calculate_sortino_ratio`, `calculate_calmar_ratio`, `calculate_treynor_ratio` et `calculate_information_ratio`.

**Test ajouté :** `tests/test_performance_metrics.py::test_beta_alpha_nan_variance` — vérifie qu'une série benchmark entièrement `NaN` retourne `(None, None)` sans explosion.

**Vérification :** 761 tests passent, dont le nouveau.
